# في هذا القسم سنشرح مشروع بسيط :

هذا نموذج مخطط ER سنقوم بتنفيذ هذا المخطط ننشئ قاعده بيانات والجدوال المطلوبة والربط بينها لكننا سنققل الاعمده نظرا لانها اعمدة المخطط كثيره

هذا المخطط SCHema اللذي سينتج بعد انشائنا للجداول وربطها ببعضها بحيث يوضح هذا المخطط نوع كل عمود ومسار الربط

اتبع الخطوات بالترتيب لتحقيق النتائج المطلوبة : ملاحظة : يمكنك تجربة اي استعلام والتدرب اكثر علي الاستعلام لاتكتفي فقط بالامثلة الموجوده

1- انشاء قاعدة بيانات جديده :

SQL (DDL)

CREATE DATABASE school;

2- تحديد قاعدة البيانات :

SQL (DDL)

USE school;

هنا نقوم باختيار قاعدة البيانات الخاصة بنا لاجراء العمليات عليها

3- انشاء جدول الاول (الاقسام ) :

SQL (DDL)

    
CREATE TABLE depr (
    id INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    num INT NOT NULL
                    );
هنا قمنا بانشاء جدول الاقسام واللذي يحتوي على عمود ال id واسم القسم ورقم للقسم

4- انشاء الجدول الثاني ( الطلاب )

SQL (DDL)

    
CREATE TABLE students(
    id INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    phone INT NOT NULL,
    num_id INT NOT NULL UNIQUE,
    age DATE NOT NULL,
    depr_id INT NOT NULL,
    FOREIGN KEY (depr_id) REFERENCES depr (id)  ON DELETE CASCADE ON UPDATE CASCADE 
                    );
هنا قمنا بانشاء جدول الطلاب واللذي يحتوي على id واسم الطالب ورقم الهاتف ورقم اكاديمي للطالب والعمر ومفتاح فرعي يشير الى جدول الاقسام

5- انشاء الجدول الثالث ( المواد )

SQL (DDL)

    
CREATE TABLE materials(
    id INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    hours INT NOT NULL,
    depr_id INT NOT NULL,
    FOREIGN KEY (depr_id) REFERENCES depr (id)  ON DELETE CASCADE ON UPDATE CASCADE 
                    );
هنا قمنا بانشاء جدول المواد واللذي يحتوي على id واسم المادة وعدد الساعات ومفتاح فرعي يشير الى جدول الاقسام

6- انشاء الجدول الرابع ( الشهائد )

SQL (DDL)

    
CREATE TABLE Results (
    id INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    avg FLOAT NOT NULL,
    code INT NOT NULL UNIQUE,
    stud_id INT NOT NULL UNIQUE,
    FOREIGN KEY (stud_id) REFERENCES students (id)  ON DELETE CASCADE ON UPDATE CASCADE 
                    );
هنا قمنا بانشاء جدول الشهائد واللذي يحتوي على id والمعدل النهائي ورقم فريد لكل شهاده ومفتاح فرعي يشير لجدول الطلاب

6- انشاء الجدول الخامس لربط الجدولين ( الطلاب مع المواد )

SQL (DDL)

    
CREATE TABLE stud_with_matr (
    stud_id INT NOT NULL,
    mat_id INT NOT NULL,
    FOREIGN KEY (stud_id) REFERENCES students (id)  ON DELETE CASCADE ON UPDATE CASCADE ,
    FOREIGN KEY (mat_id) REFERENCES materials (id)  ON DELETE CASCADE ON UPDATE CASCADE 
                    );
هنا قمنا بانشاء جدول الطلاب مع المواد واللذي يعتبر علاقة كل الى الكل واللذي يحتوي على مفتاحين فرعيين يشيران الى جدولي الطلاب والمواد

16- عرض اسماء الطلاب مع ربط جدول اخر :

SQ (DML)

SELECT students.name FROM students INNER JOIN depr ON students.depr_id = depr.id WHERE depr.id = 2;

سيتم عرض اسماء الطلاب اللذين مسجلين في القسم الثاني فقط

17- عرض اسماء المواد وعدد الساعات مع ربط جدول اخر :

SQ (DML)

SELECT materials.name, materials.hours FROM materials INNER JOIN depr ON materials.depr_id = depr.id WHERE depr.id = 1;

سيتم عرض اسماء المواد وعدد الساعات المضمنة في القسم الاول فقط

18- عرض اسم الطالب ونتيجته مع ربط جدول اخر :

SQ (DML)

SELECT students.name,results.avg FROM results INNER JOIN students ON results.stud_id = students.id WHERE students.id = 3;

سيتم عرض اسم الطالب والنتيجه الخاصة به فقط

19- عرض اسم الطالب والمادة مع ربط الجدول الوسيط :

SQ (DML)

SELECT DISTINCT students.name, materials.name FROM stud_with_matr INNER JOIN students ON stud_with_matr.stud_id = students.id INNER JOIN materials ON stud_with_matr.mat_id = materials.id;

سيتم عرض اسم الطالب واسم الماده بناءا على علاقة كل الى الكل
